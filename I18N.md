# Internationalization (i18n) Setup

## Languages Supported

- 🇬🇧 **English** (en) - Default
- 🇳🇴 **Norwegian Bokmål** (no)
- 🇩🇰 **Danish** (da)

## Usage

The app uses `react-i18next` for internationalization. Translations are automatically detected from the browser language and can be changed using the language switcher in the navbar.

### Changing Language

Click the flag icon in the navbar to select a language. The selection is persisted in localStorage.

### Adding Translations

Translation files are located in:
```
packages/client/public/locales/
├── en/translation.json
├── no/translation.json
└── da/translation.json
```

### Using Translations in Components

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t('catalog.title')}</h1>
      <p>{t('nav.hello', { name: 'User' })}</p>
    </div>
  );
}
```

### Translation Keys

All translation keys are organized by feature:
- `app.*` - App-level strings
- `nav.*` - Navigation
- `auth.*` - Authentication
- `catalog.*` - Item catalog
- `item.*` - Item details
- `dashboard.*` - User dashboard
- `admin.*` - Admin panel
- `common.*` - Common UI elements
- `errors.*` - Error messages
- `messages.*` - Success/info messages

### Adding a New Language

1. Create a new folder: `packages/client/public/locales/{language-code}/`
2. Add `translation.json` with all keys
3. Update `packages/client/src/i18n/config.ts`:
   ```typescript
   import newLangTranslation from '../../public/locales/{code}/translation.json';
   
   // Add to resources
   resources: {
     // ...
     {code}: {
       translation: newLangTranslation,
     },
   }
   ```
4. Add language to `LanguageSwitcher.tsx`:
   ```typescript
   { code: '{code}', name: 'Name', flag: '🏳️' }
   ```
